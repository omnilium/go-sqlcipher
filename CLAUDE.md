# CLAUDE.md — go-sqlcipher

`github.com/omnilium/go-sqlcipher` is Omnilium's fork of [`mattn/go-sqlite3`](https://github.com/mattn/go-sqlite3): a CGo `database/sql` SQLite driver, forked to add **SQLCipher** at-rest encryption. It is a standalone library in its own independent git repo, consumed by downstream projects as a Go module — so a change a consumer needs isn't done until a fork tag is published for them to pin.

## Build, test, lint

CGo build. A C toolchain **and OpenSSL `libcrypto` (dev headers + library)** are required at build **and** run time — the SQLCipher codec links `-lcrypto` dynamically; there is no self-contained/libtomcrypt build (evaluated and dropped). Supported targets: **Linux x86-64/ARM64 and Apple Silicon macOS only** — Intel macOS and Windows are unsupported.

Four core gates, all run by CI:

```sh
go build ./...
golangci-lint run ./...      # golangci-lint v2, config .golangci.yaml
go test -race ./...
govulncheck ./...
```

Two further gates CI runs that those four miss — run them locally too when a change touches tag-gated code, a parser/decoder boundary, or the fork's key-quoting:

- **Feature-tag matrix:** `go test -tags <tag> ./...` for each optional build tag whose test file the default build omits (tag list under *CI*). Build tags follow upstream (`sqlite_fts5`, `sqlite_vtable`, `libsqlite3`, …); see the `sqlite3_opt_*.go` files.
- **Fuzz smoke:** each native Go fuzz target in `fuzz_test.go` under `go test -fuzz` for a bounded time. Fuzz one with `go test -run '^$' -fuzz '^FuzzName$' -fuzztime=30s .`; a plain `go test` already runs every target's seed corpus as ordinary subtests.

## Upstream tracking & resync

`master` = the latest upstream **release tag** + Omnilium's patch stack. Track release tags, **never** upstream's unreleased `master` tip. There are two independent upstreams, bumped as separate operations:

- **Go layer** — base `v1.14.45`, remote `upstream` = `https://github.com/mattn/go-sqlite3.git`. Tracks the Go binding layer + public API only.
- **C amalgamation** — SQLCipher, regenerated by the `upgrade/` tool (pinned version + procedure under *Amalgamation source*, Conventions).

The patch stack that rebases forward on every resync is exactly:

- **SQLCipher integration** — the codec/crypto cgo flags (`SQLITE_HAS_CODEC`, `SQLITE_TEMP_STORE=2`, `SQLITE_EXTRA_INIT/SHUTDOWN`, `HAVE_STDINT_H`), the OpenSSL provider wiring (`sqlite3_crypto_openssl.go`, `-DSQLCIPHER_CRYPTO_OPENSSL -lcrypto`), and the `_key` + cipher-tuning DSN mechanism in `sqlite3.go`.
- **Foundational `go.mod` patches** — the module path `github.com/omnilium/go-sqlcipher` (importable under its own path, no `replace` directive), and the Go floor held at the Omnilium baseline (`go 1.26`, in step with the wider Omnilium toolchain) above upstream's lower minimum. CI reads the version via `go-version-file: go.mod`, so bumping the floor there bumps CI too.
- **Lint/refactor fixes** — see *Forked code is ours*, Conventions.

**Resync procedure:** `git fetch --tags upstream` → `git rebase --onto vX.Y.Z <old-base> master` → get every CI gate green → force-push `master` → cut the next fork tag. `master` is force-pushed on every resync — expected and correct for a vendored mirror; push directly, no protected-branch-bypass commentary.

**Versioning:** the fork has its **own independent semver line starting at `v0.1.0`**, not aligned to upstream's `v1.14.x` numbers. Consumers pin a fork **tag**, never a branch pseudo-version (a resync rewrites `master`'s SHAs and would invalidate it).

## CI

`.github/workflows/ci.yml` — six jobs, every one on a Blacksmith runner (house rule, see `conventions:writing-workflows`), Go version read from `go.mod`. Every job first provisions OpenSSL (`libssl-dev` on Linux, Homebrew `openssl` on macOS) and Go via the shared `.github/actions/setup` composite action.

- **Build** + **Test (race)** — OS/arch matrix (Linux amd64 + arm64, Apple Silicon macOS), since the cgo build and its OpenSSL linkage are platform-specific.
- **Feature tag** — turns on each optional tag whose test the default build leaves uncompiled (`sqlite_column_metadata`, `sqlite_math_functions`, `sqlite_preupdate_hook`, `sqlite_unlock_notify`, `sqlite_vtable`, `sqlite_fts5`) and runs the suite under it. Once on Linux x86-64, plain `go test` (the matrix above already covers race). `sqlite_userauth` is deliberately excluded — see Conventions.
- **Fuzz** — one leg per `fuzz_test.go` target (`go test -fuzz` fuzzes one at a time), short `-fuzztime` mutation smoke. Once on Linux x86-64.
- **Lint** + **Vulnerability scan** — platform-agnostic static analyses, once on Linux x86-64.

## Conventions & invariants

- **Forked code is ours.** Hold the **whole tree — vendored upstream Go included** — to our full lint/quality bar: fix findings, never add per-finding "not our code" exemptions. These carried lint/refactor fixes are part of the patch stack and rebase forward on every resync (the resync isn't done until the gates are green). Linting is golangci-lint v2 (`.golangci.yaml`); its only adjustments are documented, driver-intrinsic, and minimal — not blanket waivers:
  - `exhaustive` ignores `reflect.Kind` and the cgo `_Ctype_int` type — type-dispatch switches, not domain enums.
  - `gocritic`'s `dupSubExpr` is excluded — it misfires on cgo wrapper calls (`C.sqlite3_*`).
  - `revive`'s ALL_CAPS rule is excluded — the `SQLITE_*` constants mirror SQLite's C API and are public drop-in API.
  - the opt-in, deliberately-weak SHA1 crypt encoders carry `//nolint:gosec` with justification (golangci honours `//nolint`, not gosec's native `//nosec`).
  - issue caps are disabled (`max-issues-per-linter`/`max-same-issues: 0`) so nothing is silently truncated.
- **Commits:** Conventional Commits 1.0.0 (house style). Keep the patch stack small, self-contained, and clearly labeled so it rebases cleanly.
- **Copyright headers:** add Omnilium's header — `// Copyright (C) <year> OMNILIUM ADVANCED CYBERNETICS SRL. All rights reserved.` followed by the MIT-style license line — **only to files we create**, with the **current year** and the **legal entity name** (`OMNILIUM ADVANCED CYBERNETICS SRL`, not the "Omnilium" brand). Files inherited from upstream keep their existing header (or stay headerless) — never prepend ours to a file we merely modified.
- **Don't reintroduce upstream's OSS apparatus.** The fork deliberately drops mattn's `_example/`, OSS-Fuzz/Docker example workflows, `FUNDING.yml`, Codecov config, and `SECURITY.md`. Don't restore them on resync.
- **`sqlite_userauth` is a dead upstream stub — leave it alone.** SQLite's authors removed the userauth extension (never a security boundary — an app-level password check over a *plaintext* file, which SQLCipher's real at-rest encryption supersedes here), so its C code is gone from the amalgamation and upstream turned `sqlite3_opt_userauth.go` into a no-op shim whose every method returns `errUserAuthNoLongerSupported` (mattn #1341). Kept verbatim behind the opt-in tag purely for drop-in API compatibility — downstream code referencing `Authenticate`/`AuthUserAdd`/`SQLITE_AUTH`/… still compiles and fails loudly at runtime. **Don't delete it** (breaks the drop-in promise, adds permanent rebase friction), **don't try to make `sqlite3_opt_userauth_test.go` pass** (impossible by design — it asserts the not-supported error), and it stays excluded from the feature-tag CI matrix for the same reason.
- **Amalgamation source:** `sqlite3-binding.c/.h` and `sqlite3ext.h` are **SQLCipher** (Zetetic) **v4.14.0**, not vanilla `sqlite.org` — and are **generated, never hand-edited**. The `upgrade/` tool (`upgrade.go` + `upgrade.sh`) regenerates them: clone SQLCipher at the pinned `sqlcipherTag`, run `./configure && make sqlite3.c sqlite3.h`, then apply the `USE_LIBSQLITE3` noop wrapper + the `sqlite3.h`→`sqlite3-binding.h` include rewrite. To bump SQLCipher, change `sqlcipherTag` in `upgrade/upgrade.go` and run `sh upgrade/upgrade.sh` (needs `git`, `tclsh`, a C toolchain). The amalgamation carries every crypto provider gated by `-DSQLCIPHER_CRYPTO_*`; this fork compiles with `-DSQLCIPHER_CRYPTO_OPENSSL` (see `sqlite3_crypto_openssl.go`).
